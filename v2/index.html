만나 흐르는 자리

카카오톡 더보기탭은 결제·게임·날씨·뉴스까지, 각기 다른 서비스가 하나의 스크롤 안에 공존하는 허브입니다.
이 문서는 그 공존을 가능하게 하는 공통 디자인 문법을 정의합니다.

왜 공통 문법이 필요한가

더보기탭은 카카오 내외부 서비스가 동시에 노출되는 허브 지면입니다. 서비스마다 다른 디자인 결정이 혼재하면 사용자 경험이 파편화됩니다. 디자인 시스템은 그 파편화를 막는 공통 언어입니다.

10+
한 화면에 동시에 노출되는 서비스 유형
1 color
결제 진입에만 사용되는 카카오 옐로우
1 scroll
모든 서비스가 공존하는 단일 피드

3가지 원칙

공존
Coexistence

같은 성격의 정보는 16px 라운드 카드로 묶고, 서로 다른 정보 간의 관계는 12px 간격으로 구분합니다. 이질적인 서비스가 한 화면에서 충돌 없이 공존합니다.

발견성
Discoverability

카카오 옐로우 #fee500는 결제 진입에만 사용합니다. 그레이스케일 기반 화면에서 포인트 컬러는 사용자가 무엇을 먼저 봐야 하는지 알려주는 신호입니다.

신뢰
Trust

기준 뷰포트 393px, 내부 패딩 16px, 라운드 16px — 모바일 퍼스트 일관성이 결제와 금융 정보를 다루는 화면에서도 사용자 신뢰를 만듭니다.

시스템 구성

Foundation

색상·타이포·스페이싱·아이콘처럼 화면이 공유하는 디자인의 최소 단위를 토큰으로 정의합니다.

Components

칩·카드·쇼트컷·탭바처럼 화면을 구성하는 재사용 가능한 UI 블록을 컴포넌트로 정의합니다.

Patterns

앱 셸·홈 피드처럼 컴포넌트를 조합해 만드는 화면 단위의 구성 규칙을 패턴으로 정의합니다.

시작하기

01
토큰 먼저 확인하기
색상·폰트·여백은 CSS Custom Property로 정의됩니다. 새 요소를 만들 때 기존 컬러 토큰과 스페이싱 토큰을 우선 사용합니다.
02
컴포넌트로 조합하기
홈 피드는 독립 컴포넌트를 세로로 쌓은 구조입니다. 각 컴포넌트의 마진·패딩 규칙은 Patterns 문서를 참고합니다.
03
패턴으로 화면 구성하기
헤더 → 콘텐츠 → 탭바로 이뤄지는 앱 셸 구조 안에서 콘텐츠 영역만 교체합니다.
